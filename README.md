# Schedule Parser

Парсер расписания занятий из Excel-файлов (.xlsx) с загрузкой данных в MySQL.

Проект состоит из двух этапов:

1. **Нормализация** — подготовка исходных Excel-таблиц (разъединение объединённых ячеек).
2. **Парсинг** — чтение нормализованных файлов и запись расписания в базу данных.

## Требования

- Python 3.10+
- MySQL 8.0+ (или MariaDB с поддержкой utf8mb4)
- Исходные файлы расписания в формате `.xlsx`

## Структура проекта

```
schedule_parser/
├── Tables/              # Исходные Excel-файлы расписания (добавляете сами)
├── normalized/          # Нормализованные файлы (создаются автоматически)
├── NormalizeExcel.py    # Скрипт нормализации Excel
├── main.py              # Основной парсер и загрузка в БД
├── DB.py                # Работа с MySQL
├── schema.sql           # SQL-схема таблиц
├── requirements.txt     # Зависимости Python
└── README.md
```

## Установка

### 1. Клонирование репозитория

```bash
git clone https://github.com/danyalacio/schedule_parser.git
cd schedule_parser
```

### 2. Виртуальное окружение

```bash
python3 -m venv .venv

# macOS / Linux
source .venv/bin/activate

# Windows
.venv\Scripts\activate
```

### 3. Установка зависимостей

```bash
pip install -r requirements.txt
```

### 4. Настройка MySQL

Создайте базу данных и таблицы:

```bash
mysql -u root -p < schema.sql
```

Или вручную в MySQL-клиенте выполните содержимое файла `schema.sql`.

### 5. Настройка подключения к БД

Откройте файл `DB.py` и укажите параметры подключения:

```python
mydb = mysql.connector.connect(
    host="localhost",
    user="root",
    password="ваш_пароль",
    database="imsitid"
)
```

| Параметр   | Описание              | По умолчанию |
|------------|-----------------------|--------------|
| `host`     | Адрес сервера MySQL   | `localhost`  |
| `user`     | Имя пользователя      | `root`       |
| `password` | Пароль                | `""`         |
| `database` | Имя базы данных       | `imsitid`    |

## Подготовка Excel-файлов

### Формат исходных файлов

1. Поместите файлы расписания в папку `Tables/`.
2. Поддерживается формат `.xlsx`.
3. Ожидаемая структура таблицы:
   - Колонка **«Дни»** — дни недели (ПОНЕДЕЛЬНИК, ВТОРНИК и т.д.).
   - Колонка **«Часы»** — время пар (например, `08.00-09.30`).
   - Остальные колонки — названия учебных групп.
   - В ячейках расписания:
     - **Строка 1** — название предмета.
     - **Строка 2** — преподаватель и аудитория (через три пробела или в конце строки).

### Поддерживаемое время пар

| Время          | Номер пары |
|----------------|------------|
| 08.00-09.30    | 1          |
| 09.40-11.10    | 2          |
| 11.30-13.00    | 3          |
| 13.10-14.40    | 4          |
| 14.50-16.20    | 5          |
| 16.30-18.00    | 6          |
| 18.10-19.40    | 7          |

## Использование

### Шаг 1. Нормализация Excel

Разъединяет объединённые ячейки в исходных файлах и сохраняет результат в `normalized/`:

```bash
python NormalizeExcel.py
```

Для каждого файла `Tables/1.xlsx` будет создан файл `normalized/1.xlsxCourse.xlsx`.

### Шаг 2. Парсинг и загрузка в БД

```bash
python main.py
```

Скрипт выполняет:

1. Очистку таблиц `schedule`, `teacher_schedule`, `teachers`.
2. Пересоздание таблиц.
3. Чтение всех `.xlsx` / `.xls` из папки `normalized/`.
4. Парсинг занятий по группам, дням и неделям (1-я и 2-я).
5. Заполнение таблицы `schedule`.
6. Сбор уникальных преподавателей в `teachers`.
7. Формирование расписания преподавателей в `teacher_schedule`.

> **Внимание:** `main.py` при каждом запуске **полностью пересоздаёт** таблицы и удаляет прежние данные.

## Структура базы данных

### Таблица `schedule`

Расписание занятий по группам.

| Поле            | Тип          | Описание                    |
|-----------------|--------------|-----------------------------|
| `id`            | int          | ID записи                   |
| `group_name`    | varchar(255) | Название группы             |
| `week_number`   | tinyint      | Номер недели (1 или 2)      |
| `day_of_week`   | tinyint      | День недели (1=Пн … 6=Сб)   |
| `lesson_number` | tinyint      | Номер пары (1–7)            |
| `subject_name`  | varchar(255) | Название предмета           |
| `room_number`   | varchar(50)  | Номер аудитории             |
| `teacher_name`  | varchar(100) | ФИО преподавателя           |
| `start_time`    | time         | Время начала                |
| `end_time`      | time         | Время окончания             |

### Таблица `teachers`

Справочник преподавателей.

### Таблица `teacher_schedule`

Расписание преподавателей (формируется автоматически из `schedule`).

## Проверка результата

После успешного запуска `main.py` в консоли появится сообщение `Connection`, а затем лог обработки файлов.

Примеры SQL-запросов для проверки:

```sql
-- Расписание группы
SELECT * FROM schedule WHERE group_name = 'ИС-21' ORDER BY week_number, day_of_week, lesson_number;

-- Расписание преподавателя
SELECT ts.*, t.full_name
FROM teacher_schedule ts
JOIN teachers t ON t.id = ts.teacher_id
WHERE t.full_name = 'Иванов И.И.';

-- Количество записей
SELECT COUNT(*) FROM schedule;
SELECT COUNT(*) FROM teachers;
```

## Возможные проблемы

### `Connection` не появляется / ошибка подключения к MySQL

- Убедитесь, что MySQL запущен.
- Проверьте логин, пароль и имя базы в `DB.py`.
- Убедитесь, что база `imsitid` создана (`schema.sql`).

### `FileNotFoundError` или пустая папка `normalized/`

- Сначала выполните `python NormalizeExcel.py`.
- Убедитесь, что в `Tables/` есть `.xlsx` файлы.

### Дубликаты пропускаются

Парсер не добавляет повторяющиеся записи с одинаковыми `group_name`, `week_number`, `day_of_week`, `lesson_number`. В консоли будет сообщение `Дубликат пропущен`.

### Некорректный парсинг преподавателя или аудитории

Проверьте формат ячеек: вторая строка должна содержать ФИО и номер кабинета, разделённые тремя пробелами, либо номер кабинета в конце строки (например, `1-103`, `с/зал`).
